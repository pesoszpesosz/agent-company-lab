# Buyer Guide

## Who This Kit Helps

This kit helps builders turn an AI-agent workflow into a reusable skill packet. It is best for local workflows, internal operating procedures, research memos, product packets, QA checklists, and other repeatable agent tasks.

## What You Can Build With It

- A `SKILL.md` file for a narrow workflow.
- A stop-gate checklist.
- A service-request checklist for actions that need approval.
- An artifact contract that tells agents what to save.
- An acceptance review template.
- A fictional example users can adapt.

## How To Use It

1. Choose one workflow that repeats.
2. Open `templates/SKILL.template.md`.
3. Fill the trigger rules and allowed actions.
4. Add forbidden actions before writing workflow steps.
5. Attach `gate-checklist.template.md`.
6. Define the saved artifact with `artifact-contract.template.md`.
7. Test the workflow against a fictional example.
8. Review the output with `acceptance-review.template.md`.

## Design Rule

Start narrow. A good skill does one job reliably and stops when a different owner, platform, account, payment, public-action, credential, or compliance gate appears.

## What This Kit Does Not Include

- Marketplace account setup.
- Seller profile setup.
- Payment, tax, KYC, or payout setup.
- Legal advice.
- Public listing submission.
- Browser automation.
- Guaranteed sales or marketplace acceptance.

## Recommended First Skill

Build a `local-research-memo` skill first. It is useful, safe, and easy to test because the whole workflow can stay inside local files.
